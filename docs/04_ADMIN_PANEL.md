# 04. Admin Panel

## Назначение

Админка нужна для управления справочниками и просмотра заявок.

Это инструмент Олега и администратора проекта.

## Технология

Админка реализована на Fastify.

Фронтенд — server-rendered HTML.

SPA не используется.

React/Vue не используются.

## Запуск

Админка запускается вместе с ботом через:

```bash
npm run dev
```

Локальный адрес по умолчанию:

```text
http://localhost:3000/admin
```

Порт задаётся через `ADMIN_PORT`.

## Главная страница `/admin`

Показывает счётчики:

* салоны;
* менеджеры;
* заявки;
* неиспользованные коды.

Также содержит ссылки на разделы:

* `/admin/salons`;
* `/admin/managers`;
* `/admin/auth-codes`;
* `/admin/orders`.

## Салоны

Раздел:

```text
/admin/salons
```

Возможности:

* список салонов;
* создание салона;
* редактирование салона;
* управление тарифами;
* управление типом оплаты по умолчанию;
* включение фиксированной оплаты через `is_payment_by_fixed`;
* управление активностью и сортировкой.

## Обязательные позиции салона

Управление `salon_required_items` сейчас находится на странице редактирования салона.

Для салона можно добавить обязательную позицию:

* `item_type`;
* `item_name`;
* `unit_price`;
* `price_text`;
* `card_text`;
* `quantity`;
* `is_required`;
* `auto_add_to_order`;
* `comment`;
* `sort_order`;
* `is_active`.

Редактирование и отключение обязательных позиций поддержаны на уровне repository, но в UI сейчас явно реализовано добавление и отображение. При необходимости UI редактирования/отключения нужно добавить отдельной задачей.

## Менеджеры

Раздел:

```text
/admin/managers
```

Возможности:

* список менеджеров;
* создание менеджера;
* редактирование менеджера;
* привязка к салону;
* телефон;
* email;
* должность;
* роль `manager` или `director`;
* активность;
* сортировка.

## Коды доступа

Раздел:

```text
/admin/auth-codes
```

Возможности:

* создание одноразового кода для менеджера;
* просмотр списка кодов;
* просмотр использованности;
* сброс кода.

Код доступа связан с `manager_id`.

## Заявки

Раздел:

```text
/admin/orders
```

Возможности:

* список заявок;
* просмотр заявки;
* просмотр snapshot-полей;
* просмотр позиций заявки;
* просмотр готовой карточки в `<pre>`.

`final_price` в админке не показывается, потому что итоговую стоимость на MVP не рассчитывает система. В списке и деталке заявок можно показывать тарифные поля, например `base_price`, но не итоговую стоимость.

## Авторизация админки

Локально админка может работать без авторизации.

Перед production-деплоем админку нельзя открывать наружу без защиты.

Минимальные варианты защиты для MVP:

* Basic Auth на nginx;
* доступ только через VPN;
* доступ только через SSH tunnel;
* закрытый firewall.

## Что не делать в админке на MVP

Не добавлять без необходимости:

* React/Vue;
* отдельную сборку фронтенда;
* сложную дизайн-систему;
* полноценную CRM;
* сложные роли и ACL.
